# 設定 Ollama(本機執行)

## 這是什麼

Ollama 讓你在**自己的電腦上**執行開源 LLM,**完全免費、不需金鑰、不需信用卡**。所有問題與資料摘要**只在你的電腦上處理,不會傳送到任何外部伺服器**——是本模組隱私保護最佳的選項,適合處理敏感資料時使用。代價是回覆品質與速度取決於你的電腦效能與所選模型大小。

## 安裝與準備

1. 前往 <https://ollama.com> 下載並安裝 Ollama(支援 Windows/macOS/Linux)。
2. 安裝完成後,Ollama 會在背景以 `http://localhost:11434` 提供服務。
3. 開啟終端機(PowerShell 或命令提示字元),下載一個模型,例如:

   ```powershell
   ollama pull llama3.2
   ```

   下載時間依模型大小與網路速度而定,僅需一次。

## 設定金鑰

**不需要**——Ollama 在本機執行,不需要任何 API 金鑰。這是五個 provider 中唯一免金鑰的選項。

## 在 askLLM 中使用

- 確認 Ollama 已啟動(安裝後通常會自動啟動並常駐背景;若不確定,可在終端機執行 `ollama list` 確認能否連線)。
- **Provider** 下拉選單選「Ollama (local)」。
- **Model** 欄位預設為 `llama3.2`;若你 `pull` 的是其他模型(如 `mistral`、`qwen2.5`),請改填該模型名稱,需與 `ollama pull` 時的名稱一致。

## 常見問題

| 畫面訊息 | 代表意義 | 處理方式 |
|---|---|---|
| 端點或模型名錯誤(model: ...) | Model 欄位的名稱本機沒有安裝 | 執行 `ollama pull <模型名稱>`,或執行 `ollama list` 確認已安裝的模型名稱 |
| 無法連線,請檢查網路 | Ollama 服務未啟動,或 `localhost:11434` 被防火牆擋住 | 確認 Ollama 應用程式正在執行;必要時重新啟動 Ollama |
| 已達用量上限,稍後再試 | 極少見(本機執行通常不會有此限制) | 檢查是否有其他程式同時大量佔用 Ollama |

## 為什麼選 Ollama

如果你分析的是不能外流的敏感資料(如未公開的臨床或組織資料),Ollama 是本模組**唯一零資料外送**的選項:所選變項的摘要統計不會離開你的電腦。缺點是需要一定的硬體資源(建議至少 8GB 以上可用記憶體,依模型大小而定),且模型品質通常不如雲端大型模型。
