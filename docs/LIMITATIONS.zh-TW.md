# 限制與使用建議

askLLM 讓 LLM 讀你的資料摘要並提出統計建議,但 **LLM 會產生看似合理卻錯誤的內容(幻覺)**。本頁記錄實測觀察到的錯誤類型,以及據此建議的使用方式。

實測方式:以 `tools/compare-models.R` 對 iris 與 mtcars 各問一題,比較 GitHub Models 上的 `openai/gpt-4o-mini`、`openai/gpt-4.1`、`openai/gpt-4.1-mini`、`microsoft/phi-4`(2026-07-21)。

## 觀察到的錯誤類型

### 1. jamovi 選單路徑最常出錯(所有受測模型都犯)

這是**最需要警覺**的一類:模型能正確判斷「該做 ANOVA」,卻把操作路徑寫錯,而且語氣同樣肯定。實測出現的錯誤路徑:

| 模型寫的路徑 | 問題 |
|---|---|
| `分析 > 比較 > 獨立樣本t檢定` | jamovi 沒有「比較」選單(t 檢定在 **T-Tests**) |
| `探索 > 相關性` | 相關矩陣在 **Regression**,不在 Exploration |
| `探索 > 主成分分析` | PCA 在 **Factor** 選單 |
| `分類 > 判別分析`、`機器學習 > 分類器` | **jamovi 內建根本沒有這些選單與分析** |
| `計算 > 描述性統計`、`視覺 > 分類圖形`、`預測 > 方差分析` | 選單名稱全屬虛構 |
| `Regression > General Linear Model > Linear Regression` | 巢狀層級是編的 |
| `迴歸 → 線性線性(Linear Regression)` | 連中文標籤都出現錯字 |

**建議**:把回覆中的分析**名稱**當作起點,**路徑**一律以 jamovi 實際介面為準。教學時可刻意用這點示範「AI 的自信與正確性無關」。

### 2. 統計建議大致合理,但需自行判斷適用性

受測模型對 iris 建議描述統計、ANOVA、相關、PCA,對 mtcars 建議多元迴歸——方向都對。但它們**看不到你的研究問題**,只看得到摘要統計,因此:

- 不會知道你的資料是否符合分析假設(常態、獨立、變異數同質)
- 不會知道變項的實際意義與測量方式
- 可能建議與研究目的無關的分析(例如純探索性的 PCA)

**建議**:把回覆當成「可能的分析清單」,再用你自己的研究問題篩選。假設檢查請用 jamovi 內建的 Assumption Checks,不要問 LLM。

### 3. 回應速度差異極大

| 模型 | 實測耗時 |
|---|---|
| `openai/gpt-4.1` | 4.7 秒 |
| `openai/gpt-4.1-mini` | 5.4 秒 |
| `openai/gpt-4o-mini` | 5.7–10.2 秒 |
| `microsoft/phi-4` | **314–315 秒(逾 5 分鐘)** |

**建議**:課堂示範避開 `microsoft/phi-4` 這類慢速模型。若送出後久候未回,多半是模型本身慢,不是當掉——等待訊息會一直顯示。

### 4. 模型目錄與實際可用不一致

`meta/meta-llama-3.1-8b-instruct` 出現在 GitHub Models 目錄中,實際呼叫卻回 **HTTP 400 `Unknown model`**。同系列的 `meta/llama-3.3-70b-instruct` 則正常。

**建議**:換模型後先問一個簡單問題確認可用,再拿來做正式比較。

## 綜合使用建議

1. **把 askLLM 當「腦力激盪的起點」,不是「統計顧問的結論」。** 它擅長把你的資料特徵翻譯成可能的分析方向,不擅長給出可直接照做的操作步驟。
2. **選單路徑一律自行核對。** 這是實測中最穩定的錯誤來源。
3. **勾選 Attach data summary。** 不勾選時模型完全在猜(實測曾憑空生出「8 個樣本、Saguaro/Palo Verde 三個物種」這種與資料無關的內容)。
4. **對統計初學者使用時,請搭配教師說明。** 學生難以分辨「路徑錯誤」與「分析建議錯誤」的差別。
5. **交叉比對兩個以上模型。** 若兩個模型對同一份資料給出一致的分析建議,可信度較高;路徑則仍需核對。
6. **不要用它取代假設檢查與結果解讀。** jamovi 的 Assumption Checks 與實際輸出才是依據。

## 給教學使用的建議設計

這些限制其實是**很好的教材**:

- 讓學生先問 askLLM,再實際到 jamovi 操作,親身發現路徑錯誤 → 具體體會「LLM 幻覺」
- 用 `tools/compare-models.R` 比較不同模型的回答,討論為何有的完整、有的簡略
- 討論「為什麼摘要統計足以建議分析,卻不足以下結論」
