## 1. 背景與動機

### 1.1 實測幻覺:選單路徑是最穩定的錯誤來源

`docs/LIMITATIONS.zh-TW.md` 記錄的實測(2026-07-21,`tools/compare-models.R`,對 iris 與 mtcars 各一題,比較 GitHub Models 上四個模型)顯示:**所有受測模型都編造 jamovi 選單路徑**,且語氣與正確答案同樣肯定。實例包括:

- `分析 > 比較 > 獨立樣本t檢定` — jamovi 沒有「比較」選單(t 檢定在 **T-Tests**)
- `探索 > 相關性` — 相關矩陣在 **Regression**,不在 Exploration
- `分類 > 判別分析`、`機器學習 > 分類器` — **jamovi 內建根本沒有這些選單與分析**
- `Regression > General Linear Model > Linear Regression` — 巢狀層級是編的

模型能正確判斷「該做 ANOVA」,卻把操作路徑寫錯——這是 v1.0 對統計初學者最危險的失效模式。

### 1.2 Jonathon 的效益質疑

jamovi 專案負責人 Jonathon Love 暫緩收錄 askLLM 進官方 library(`dev-notes/library-submission-response.zh-TW.md`),核心效益質疑:模組只能取得變項摘要,「使用者自己把摘要數字貼進聊天視窗也能得到類似結果」,效益不足以合理化新增的資料外流風險。

### 1.3 v1.1 的回應

v1.1 讓 askLLM 掃描本機 jamovi **實際安裝的所有模組**,把真實的選單樹(installed catalog)連同官方 jamovi library 的未安裝模組清單(available list)一併餵給 LLM,並以 prompt 指令約束模型**只從真實路徑中選**:

1. **治幻覺**:回覆中的每條選單路徑都來自掃描到的真實清單,直接消除 §1.1 的最大錯誤類別。
2. **回應效益質疑**:「這台機器上安裝了哪些模組、每個模組提供哪些分析與選單路徑」是使用者無法輕易手貼、唯有模組內整合才能自動取得且隨安裝狀態即時更新的資訊——這正是「貼摘要數字」做不到的部分。
3. **附帶價值**:已安裝清單無合適分析時,從官方 library 清單指名建議安裝(Modules > jamovi library),不編造模組名。

## 2. 功能敘述

### 2.1 使用者視角

使用者在 jamovi 中開啟 Ask LLM、選變項、輸入問題、勾選 Submit——操作流程與 v1.0 完全相同。差別在回覆品質:LLM 建議的每一條 `Analyses > …` 選單路徑都真實存在於使用者當下的 jamovi 安裝中,照著點就找得到;若使用者的問題需要未安裝的分析(例如未裝 medmod 時問中介分析),回覆會明說「目前未安裝」並從官方 library 指名可安裝的模組。選項面板新增一個核取方塊「Include installed modules」(預設勾選),取消勾選即完全停用此功能並回到 v1.0 行為。

### 2.2 系統行為

`.runInner()` 在組 payload 階段,若 `includeCatalog` 選項為 TRUE:

1. 呼叫 `scan_modules()` 掃描本機 jamovi 模組目錄(目錄由 `default_module_dirs()` 多信號探索),得到已安裝模組與其分析選單結構;
2. 以 `catalog_text()` 轉為決定性純文字(字元預算 2500);
3. 讀取隨套件安裝的 `inst/catalog/known-modules.yaml`(官方 library 清單快照),以 `available_text()` 產出「官方有、本機未裝」的模組清單文字(字元預算 900);
4. 兩段文字經 `ask_llm()` → `build_prompt()` 注入 user prompt 的 `<installed_analyses>` 與 `<available_modules>` 區塊,system prompt 附加行為指令;
5. catalog 文字納入防抖快取指紋(payload 第五欄),安裝狀態或開關改變即重新呼叫。

任一環節失敗(目錄不存在、yaml 壞損、known-modules 遺失)都**靜默降級**:對應區塊為 NULL,prompt 退回 v1.0 逐字相同的形式,絕不 `stop()`、絕不在畫面出現紅字。

## 4. 介面規格

### 4.1 新檔 `R/module-catalog.R`(純函式,比照 `data-summary.R` 模式)

共同契約:**決定性**(同輸入 byte-identical 同輸出,與 locale/時區無關)、**永不 `stop()`**(一切失敗以回傳值表達)、**可注入**(路徑/資料皆可由參數覆寫供測試)。所有字元計數一律使用既有 `count_chars()`(`nchar(type = 'chars')`)。

#### 4.1.1 `default_module_dirs()`

```r
default_module_dirs <- function()
```

- **參數**:無。
- **回傳**:`character` 向量(可為 `character(0)`),為候選 jamovi 模組根目錄,已去重、僅含實際存在的目錄。
- **收集順序**(即回傳順序;去重時保留首次出現):
  1. **自我定位(主信號)**:`p <- system.file(package = 'askLLM')`。若 `p` 非空且 `basename(dirname(p)) == 'R'` 且上溯三層後 `basename(...) == 'modules'`,則收入該三層上溯結果(`dirname(dirname(dirname(p)))`)。條件不符(如開發機一般 library 安裝)→ 此信號**不產生路徑且不報錯**。
  2. **環境變數**:`JAMOVI_MODULES_PATH` 非空時,以 `.Platform$path.sep` 切分為多段;每段若為相對路徑且 `JAMOVI_HOME` 非空,則以 `file.path(JAMOVI_HOME, 段)` 解析;絕對路徑直接使用。
  3. **平台慣例路徑**:Windows `file.path(Sys.getenv('APPDATA'), 'jamovi', 'modules')`;macOS `~/Library/Application Support/jamovi/modules`;Linux `~/.jamovi/modules` 與 flatpak 變體 `~/.var/app/org.jamovi.jamovi/data/jamovi/modules`。僅收集當前平台的路徑。
  4. **`.libPaths()` 後備**:每個 lib path 若符合正規表達式 `[/\\]modules[/\\][^/\\]+[/\\]R$`,取其上溯兩層(即 `…/modules`)收入。
- **正規化與過濾**:全部路徑經 `normalizePath(winslash = '/', mustWork = FALSE)` 後 `unique()`(保序),再以 `dir.exists()` 過濾。
- **錯誤行為**:任何信號取得失敗(env var 未設、`system.file` 空字串等)一律靜默略過;函式本身永不 error。

#### 4.1.2 `scan_modules(dirs = NULL)`

```r
scan_modules <- function(dirs = NULL)
```

- **參數**:`dirs` — 模組根目錄向量;`NULL` 時使用 `default_module_dirs()`(測試以此參數注入 fixture 路徑)。
- **回傳**:`list(modules = <list>, errors = <character>)`。
  - `modules`:list,每元素為一個模組:

    ```r
    list(
      name     = <character(1)>,      # jamovi.yaml 的 name
      title    = <character(1)>,      # jamovi.yaml 的 title;缺漏時以 name 代
      analyses = list(                # 過濾後的分析,保持 yaml 宣告順序
        list(name         = <chr(1)>,
             menuGroup    = <chr(1)>,
             menuSubgroup = <chr(1) 或 NULL>,
             menuTitle    = <chr(1)>,
             menuSubtitle = <chr(1) 或 NULL>),
        ...))
    ```

  - `errors`:`character` 向量(可為 `character(0)`),每元素格式 `"<檔案路徑>: <錯誤訊息>"`。
- **掃描規則**:
  1. 對每個 dir,以 `sort(list.dirs(dir, recursive = FALSE), method = 'radix')` 取一層子目錄(radix 排序確保 locale 無關的決定性)。
  2. 子目錄含 `jamovi.yaml` 者,以 `yaml::read_yaml()` 讀取,整段包 `tryCatch`;讀取或解析失敗 → 記入 `errors`、跳過該模組、繼續掃描。
  3. **分析層過濾**:`tolower(menuGroup) == 'hidden'` 的分析剔除;缺 `menuGroup` 或缺 `menuTitle` 的分析靜默剔除。
  4. **模組層過濾**:過濾後 `analyses` 為空的模組整個剔除(不記 error;此為正常情形,如 `analyses: []` 的資源型模組)。缺 `name` 的模組記入 `errors` 後剔除。
  5. **去重**:模組 `name` 以 `tolower()` 比對,重複時**首次出現者勝**(dirs 順序即優先序),後續同名靜默略過。
- **錯誤行為**:dirs 全部不存在或全部讀取失敗 → 回傳 `list(modules = list(), errors = <累積訊息>)`;永不 `stop()`。

#### 4.1.3 `catalog_text(catalog, char_budget = 2500)`

```r
catalog_text <- function(catalog, char_budget = 2500)
```

- **參數**:`catalog` — `scan_modules()` 的回傳值(或其 `$modules` 同構 list;實作以 `catalog$modules` 存取);`char_budget` — 字元預算。
- **回傳**:`character(1)` 純文字,或 **`NULL`**(當 `catalog` 為 `NULL`、無 `$modules`、或 `modules` 為空 list 時)。回傳 `NULL` 即代表上游應完全省略 catalog 區塊(v1.0 降級)。
- **格式與排序**:見 §5.1(規範性)。
- **錯誤行為**:輸入結構異常(非 list 等)→ 回傳 `NULL`;永不 `stop()`。

#### 4.1.4 `available_text(known, installed_names, char_budget = 900)`

```r
available_text <- function(known, installed_names, char_budget = 900)
```

- **參數**:
  - `known` — `known-modules.yaml` 經 `yaml::read_yaml()` 的解析結果(含 `$source_url`/`$retrieved_date`/`$modules`;實作以 `known$modules` 存取);
  - `installed_names` — 已安裝模組名 `character` 向量(來自 `scan_modules()` 結果的 `name` 欄);
  - `char_budget` — 字元預算。
- **行為**:自 `known$modules` 中剔除 `tolower(name)` 出現在 `tolower(installed_names)` 者,餘者依 §5.2 格式輸出。
- **回傳**:`character(1)`,或 **`NULL`**(當 `known` 為 `NULL`/無 `$modules`/`modules` 為空、或剔除後無剩餘模組時)。
- **錯誤行為**:輸入結構異常 → `NULL`;永不 `stop()`。

### 4.2 `R/llm-adapter.R` 的參數擴充

#### 4.2.1 `build_prompt`

v1.0 簽名 `build_prompt(question, summary_text = NULL)` 擴充為:

```r
build_prompt <- function(question, summary_text = NULL,
                         catalog_text = NULL, available_text = NULL)
```

- **降級保證(規範性)**:`catalog_text` 為 `NULL` 或空字串時,**忽略 `available_text`**,輸出與 v1.0 `build_prompt(question, summary_text)` **逐字(byte-identical)相同**。
- `catalog_text` 非空時的組版格式見 §5.3(規範性)。
- `available_text` 區塊**只在 `catalog_text` 區塊存在時**才可能出現(架構理由:available 依賴掃描結果排除已安裝;掃描全失敗即整體降級)。

#### 4.2.2 `ask_llm`

v1.0 簽名擴充為(新增兩參數,置於 `summary_text` 之後、既有具名參數不動):

```r
ask_llm <- function(question, summary_text = NULL,
                    catalog_text = NULL, available_text = NULL,
                    base_url, model, api_key,
                    system_prompt = NULL, max_tokens = 1024, ctor = NULL)
```

- 行為:將 `catalog_text`/`available_text` 原樣透傳給 `build_prompt()`;其餘契約(回傳 `list(ok, text, model, elapsed_s, error)`、永不 `stop()`)不變。

### 4.3 `R/askllm.b.R` 的變更

#### 4.3.1 新選項 `includeCatalog`

- `jamovi/askllm.a.yaml`:`name: includeCatalog`,`type: Bool`,`default: true`,`title: Include installed modules`。`jamovi/askllm.u.yaml` 對應加入核取方塊。
- `.runInner()`:`isTRUE(opt$includeCatalog)` 時執行掃描與 catalog/available 組裝;`FALSE` 時**完全不呼叫** `scan_modules()`(不只是丟棄結果),catalog/available 皆為 `NULL`。

#### 4.3.2 payload 指紋第五欄

v1.0 簽名 `.askllm_build_payload(question, summary_text, base_url, model)` 擴充為:

```r
.askllm_build_payload <- function(question, summary_text, base_url, model,
                                  context_text = '')
```

- 第五欄 `context_text` 預設空字串——**既有測試以四參數呼叫時輸出與 v1.0 逐位相同**,舊測試不動。
- `.runInner()` 傳入值:catalog 與 available 皆為 `NULL` 時傳 `''`;否則傳 `paste(catalog %||% '', available %||% '', sep = '\n')`。
- 分隔仍以 `.ASKLLM_SEP`(U+001F)串接五欄。效果:catalog 內容改變、includeCatalog 開關切換、known-modules 更新,任一者皆使 payload 改變 → `.askllm_decide()` 回 `'call'` 而非 `'cached'`。

#### 4.3.3 system prompt

`.askllm_system_prompt()` 擴充為:

```r
.askllm_system_prompt <- function(has_catalog = FALSE)
```

- `has_catalog = FALSE`(預設):回傳字串與 v1.0 逐字相同(降級保證涵蓋 system prompt)。
- `has_catalog = TRUE`:於 v1.0 字串之後(同樣以單一空白接續)附加下列英文句(逐字,規範性):

  > `When a list of installed analyses is provided, recommend analyses ONLY from that list and cite each menu path exactly as written. If nothing installed fits, suggest a module ONLY from the provided available-modules list. Never invent module names, menus, or menu paths.`

- `.runInner()` 以 `has_catalog = !is.null(catalog_text_value)` 呼叫。

#### 4.3.4 引導文字新隱私句

`.askllm_guide_text()` 的「隱私提醒 / Privacy」段,於現有中英文句之後**各追加一句**(逐字定稿見 §9)。

#### 4.3.5 known-modules 載入

`.runInner()` 以 `system.file('catalog', 'known-modules.yaml', package = 'askLLM')` 定位、`yaml::read_yaml()` 讀取,整段包 `tryCatch`;檔案不存在或解析失敗 → `known` 為 `NULL` → `available_text()` 回 `NULL` → 無 `<available_modules>` 區塊,其餘功能不受影響。

### 4.4 `tools/sync-known-modules.R`

離線開發工具(不隨套件安裝、不在執行期使用),職責:自官方來源生成 `inst/catalog/known-modules.yaml`。

- **主函式**:

  ```r
  sync_known_modules(
      source_url     = 'https://raw.githubusercontent.com/jonathon-love/jamovi-library/master/modules.yaml',
      fetch          = NULL,          # function(url) -> character(1);NULL 時用內建下載。測試注入 fixture 讀取器
      dest           = 'inst/catalog/known-modules.yaml',
      retrieved_date = format(Sys.Date()))   # 測試注入固定日期以驗證決定性
  ```

- **輸入**:
  1. 第一段:`source_url` 指向的 `modules.yaml`(欄位 `name`/`url`/`commit`/`subdir?`/`platforms?`/`hidden?`)。
  2. 第二段:對每個未標 `hidden: true` 的條目,抓取 `https://raw.githubusercontent.com/<owner>/<repo>/<commit>/[<subdir>/]jamovi/0000.yaml`(owner/repo 自 `url` 解析)取 `title`/`description`/`analyses`。
- **輸出**:寫入 `dest` 的 UTF-8 yaml,schema 見 §4.6。
- **決定性要求(規範性)**:
  - 同一組輸入(含固定 `retrieved_date`)重跑任意次,輸出 **byte-identical**(零 diff)。
  - 條目順序:依 `name` 的 radix 排序(不依來源順序,杜絕上游洗牌造成假 diff)。
  - `provides` 生成規則:取 `description` 去除 HTML 標籤(`gsub('<[^>]+>', '', …)`)、壓縮連續空白為單一空白、`trimws()` 後,截至第一個句號(`. ` 或字串尾)為止;結果超過 160 字元時截為 159 字元 + `…`。
  - `tags` 生成規則:該模組 `analyses` 中所有非 hidden 的 `menuGroup` 值,`tolower()` 後 `unique()`、radix 排序。無分析者 `tags: []`。
- **容錯(規範性)**:任一模組的第二段抓取失敗、或 `title`/`description` 缺漏 → 以 `warning()` 記錄該模組名後**跳過該筆**,不中斷整體;第一段抓取失敗才屬致命(工具腳本允許此時 `stop()`——這是 v1.1 中唯一允許 stop 的元件,因其不在 jamovi 執行期)。

### 4.5 `tools/release-check.R` freshness 檢查

新增檢查項:

```r
check_known_modules_freshness(path = 'inst/catalog/known-modules.yaml',
                              today = Sys.Date(), threshold_days = 60)
```

- 讀取 `retrieved_date`;`as.integer(today - as.Date(retrieved_date)) > threshold_days` 時輸出一行以 `WARN:` 開頭、含 `sync-known-modules` 字樣的訊息(提示重跑同步腳本);未超過則輸出 `OK` 項。檔案缺失或日期不可解析 → 同樣輸出 `WARN:` 行。永不 `stop()`(release-check 是提醒閘,不是硬閘)。
- `today` 參數存在即為 S16 的可測性注入點。

### 4.6 `inst/catalog/known-modules.yaml` schema(規範性)

```yaml
source_url: https://raw.githubusercontent.com/jonathon-love/jamovi-library/master/modules.yaml
retrieved_date: '2026-07-23'        # ISO 8601 YYYY-MM-DD,加引號防 yaml 轉型
modules:
  - name: scatr                      # 必要;全檔唯一(不分大小寫)
    title: Scatter Plots             # 必要;非空字串
    provides: Scatter plots with marginal densities and box plots.   # 必要;一句摘述,≤160 字元
    tags: [descriptives]             # 必要;字串序列,可為空 [];值為小寫 menuGroup
```

驗證規則(S14 測試依此逐條檢查):檔頭 `source_url` 為非空字串、`retrieved_date` 符合 `^\d{4}-\d{2}-\d{2}$`;`modules` 為非空序列;每筆四鍵齊備;`name` 經 `tolower()` 後無重複;`provides` 非空且 `count_chars(provides) <= 160`;`tags` 為 character 向量(長度可為 0)。

## 5. 文字格式規格(規範性;測試逐字比對)

### 5.1 `catalog_text` 輸出格式

**模組區塊**——每個模組輸出:

```
<name> (<title>):
  Analyses > <menuGroup> > <menuTitle>
  Analyses > <menuGroup> > <menuSubgroup> > <menuTitle>
  Analyses > <menuGroup> > <menuTitle> — <menuSubtitle>
```

逐條規則:

1. **首行**:`<name>` + `' ('` + `<title>` + `'):'`。無其他裝飾。
2. **分析行**:每個分析一行,前導**兩個空白**,各段以 `' > '`(空白-大於-空白)連接:固定字面 `Analyses`、`menuGroup`、`menuSubgroup`(**僅在存在且非空時**插入)、`menuTitle`;`menuSubtitle` 存在且非空時,行尾接 `' — '`(空白-em dash U+2014-空白)+ `menuSubtitle`。
3. **分析行順序**:保持 `jamovi.yaml` 內 `analyses` 的宣告順序(即 jamovi 選單實際順序),不重排。
4. **模組區塊順序**:`name` 為 `jmv`(不分大小寫)的模組**永遠排第一**;其餘依 `order(name, method = 'radix')` 升冪(byte 序,locale 無關)。
5. **區塊連接**:區塊之間以一個空行分隔(`paste(blocks, collapse = '\n\n')`)。輸出首尾無多餘空行、無尾隨換行。
6. **budget 截斷**:以 `count_chars()` 計總長。全文 ≤ `char_budget` → 原樣輸出。否則:
   a. 依 §5.1.4 的順序**貪婪納入整個模組區塊**:逐塊試加,若「已納入區塊 + 本塊 + 截斷尾標」以 `'\n\n'` 連接後總長 ≤ `char_budget` 則納入,否則本塊與其後全部略去(**不納入半個區塊**)。
   b. **jmv 保底**:排序後第一個區塊(即 jmv;jmv 未安裝時為 radix 首位模組)**無條件納入**,即使它單獨已超過 budget(此為唯一允許超出 budget 的情形)。
   c. 略去 N ≥ 1 個模組時,於文末以 `'\n\n'` 接**截斷尾標**(逐字):

      ```
      [+N more modules omitted]
      ```

      其中 `N` 為十進位整數(如 `[+7 more modules omitted]`)。

### 5.2 `available_text` 輸出格式

**條目行**——排除已安裝後,每個模組一行:

```
- <name> (<title>): <provides>
```

逐條規則:

1. 行首字面 `'- '`,接 `<name>` + `' ('` + `<title>` + `'): '` + `<provides>`。
2. **順序**:依 `order(name, method = 'radix')` 升冪。無 jmv 特例(jmv 恆為已安裝)。
3. 行間以單一 `'\n'` 連接;無尾隨換行。
4. **budget 截斷**:比照 §5.1.6——整行為單位貪婪納入;排序首行無條件保底;略去 N ≥ 1 筆時文末以 `'\n'` 接**截斷尾標**(逐字):

   ```
   [+N more — browse the full jamovi library]
   ```

   (`—` 為 em dash U+2014,前後各一空白;`N` 為十進位整數。)

### 5.3 `build_prompt` 組版格式

以下 `{X}` 為代入變數。**情形 A(catalog 為 NULL/空)**:與 v1.0 逐字相同——

- 有 summary:

  ```
  Here is a summary of the dataset:
  <summary>
  {summary_text}
  </summary>

  Answer the user question about THIS dataset. Be concise.

  Question: {question}
  ```

- 無 summary:輸出即 `{question}` 本身。

**情形 B(catalog 非空、available 為 NULL)**:

```
{summary 段(有 summary 時,同情形 A 的前三行 + 空行;無 summary 時整段連同其後空行省略)}
Installed jamovi analyses on this machine (real menu paths):
<installed_analyses>
{catalog_text}
</installed_analyses>

Answer the user question about THIS dataset. Be concise.
Recommend analyses ONLY from <installed_analyses> and quote each menu path EXACTLY as written there.
If no installed analysis fits the question, say so plainly. NEVER invent module names or menu paths.

Question: {question}
```

**情形 C(catalog 非空、available 非空)**:於情形 B 的 `</installed_analyses>` 之後、空行之後,插入:

```
Official jamovi library modules NOT currently installed:
<available_modules>
{available_text}
</available_modules>
```

(其後接空行再接指令段。)且指令段第三行改為(取代情形 B 的第三行,逐字):

```
If no installed analysis fits, suggest installing a module ONLY from <available_modules> (Modules > jamovi library in jamovi). If neither list has a suitable option, say plainly that you do not know. NEVER invent module names or menu paths.
```

組版細則:各「段」內部如上逐行;段與段之間恰一個空行;`Question: {question}` 恆為最末行;全文無尾隨換行。summary 段的存在與否不影響 catalog/available 段格式。

## 7. 非目標(v1.1 明確不做)

- **不讀取 jamovi Results**(分析輸出):平台尚未開放模組取得分析結果;此為 jonathon 團隊的 roadmap 項目,非本模組可解。
- **不自動安裝模組**:available 清單只用於「建議使用者自行到 Modules > jamovi library 安裝」,模組本身不觸發任何安裝動作。
- **無遙測**:不收集、不回傳任何使用統計。
- **不掃描 options 細節**:catalog 只到「分析 + 選單路徑」層級,不解析各分析的 `.a.yaml` 選項結構。
- **curated assumptions 層(假設檢查知識庫)不入 v1.1**:見 §8。

## 8. Future work(v1.2 展望,非本版承諾)

**curated assumptions 檢索式注入**:v1.1 評估後擱置的假設檢查知識層(源自 jmvmcp KB 構想)。擱置理由:全量注入 token 成本高,且 §1.1 實測幻覺集中在選單路徑層、不在統計假設層。v1.2 構想:以 `question_patterns` 對使用者問題做關鍵詞/型態檢索,每次只附 1–2 筆最相關的假設檢查指引(如「t 檢定 → 常態性與變異數同質,jamovi 的 Assumption Checks 位置」),維持 token 預算不變。屆時另立規格。

其他候選:jamovi 開放分析輸出 API 後的「解讀剛跑出的結果」(價值天花板突破點,見 library-submission-response);協助模組開發者優化 Results 輸出設計。

## 9. 隱私影響

### 9.1 新增外送的資訊

v1.1 新增傳送給 LLM 服務的內容為:**已安裝 jamovi 模組的名稱、標題與選單路徑清單**,以及**官方 library 未安裝模組的名稱與一句摘述**。此為**環境中繼資料(environment metadata)**——描述軟體安裝狀態,不含任何資料集內容、變項值、檔名或使用者身分。風險評估:與 v1.0 已外送的摘要統計相比,增量風險極低;唯一新揭露是「使用者裝了哪些統計模組」,間接暗示其分析領域。

- `includeCatalog` 選項(預設開)可一鍵停用,停用後行為與 v1.0 逐位相同。
- 選 Ollama(本機)時一如既往,包含 catalog 在內的所有內容都不離開本機。

### 9.3 引導文字新隱私句(定稿,S9 逐字驗證標的)

於 `.askllm_guide_text()` 隱私段,現有中文句後追加(逐字):

現有英文句後追加(逐字):

> To ground suggestions in real menus, the NAMES AND MENU PATHS of your installed jamovi modules (environment metadata, none of your data) are also sent; untick "Include installed modules" to disable this.
