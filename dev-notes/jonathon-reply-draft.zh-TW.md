主旨:Re: askLLM——認真接下了你的價值提問,這是後續

Jonathon 你好:

謝謝你用心的回覆, 特別是那句「使用者自己把數字貼進聊天視窗也能得到差不多的結果」給我更具體的啟發，我決定目前定位這個模組為駐在jamovi內的統計方法諮詢師。

之前我嘗試讓不同LLM(GPT-4o-mini、GPT-4.1 系列、Gemini、Phi-4)根據資料檔給統計方法建議及jamovi操作順序，方法的建議大致合適，但是一個模型都會編造 jamovi 選單路徑，不只是位置寫錯，還建議 jamovi 根本沒有的功能，因此我決定往「模組做得到、貼聊天視窗做不到」的事進行迭代。迭代後的1.1版本能掃瞄使用者本機實際安裝的模組，重建真實選單樹附進 prompt,並嚴格指示模型只能從清單中推薦。使用者沒裝的分析,只能從官方 library 清單建議安裝,否則明說不知道。

測試報告在 github repo,也許能幫助你們思考模組整合 LLM 的安全建議;等你們著手定義這類模組的審查流程時,我也樂意當測試案例。

另外有一件想聽聽你意見:我在考慮透過社群分享 askLLM,邀請有興趣的 jamovi 模組開發者一起迭代,也許能採集更多有益的回饋。這個想法呼應你說的「把 AI 整合做對」,所以我想先問過你、而不是逕自往前走。你覺得如何?後續有什麼新收穫,我會再與你分享。
